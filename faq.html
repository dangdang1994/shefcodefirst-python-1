Frequently Asked Questions

General Troubleshooting

Question Answer
Why do I get the following error when I tried to run my Python code by typing python hello.py? 
python: can't open file 'hello.py':
[Errno 2] No such file or directory

Your terminal is probably not pointing at the right folder, where your Python file is saved.

To fix that, work out which folder you saved your Python code at and change the folder the terminal is pointing at by typing cd <name_of_folder>, replacing <name_of_folder> with the actual name of the folder you saved your Python code at.

Why does every time I run my code by typing python hello.py in the terminal (where hello.py is the name of my Python file) I get the following error? 
File "<stdin>", line 1
  python hello.py
             ^
SyntaxError: invalid syntax

You're most likely still running Python in interactive mode, which is a program on its own and as you can only run one program at a time in your terminal, you won't be able to run your Python file at the same time.

To fix that, type exit() or quit() to get out of interactive mode first before trying to run your Python file again.

How do I know I am running Python in interactive mode? If you see >> or In [x]: (for IPython) at the beginning of the current line in your terminal, you are in interactive mode.
(Mac/Linux only) Why does typing python -V still says I'm running Python 2.7, even though I've installed Python 3.6?

On Mac/Linux computers, Python is pre-installed into the operating system. As such, its version of Python (2.7) is found before the Python version you've downloaded.

To run your cdoe with Python 3.6 you've installed, try python3.6 hello.py instead of python hello.py.

(Windows only) What should I do if git bash freezes every time I try to run Python? Try closing git bash first, open it again then run Python. If that doesn't fix the issue, search for cmd or Powershell, open that and run Python in it instead. If none of that works, speak to an instructor to make sure the python executable is properly registered to be used directly in the command line.
(Windows only) What is the difference between git bash and GitKraken?

They are both softwares for interacting with git, the version control system you will be using throughout this course to backup and work on code together.

Git bash allows you to interact with git through the command prompt/Powershell, whereas GitKraken allows you to do the same thing through a pretty and more friendly graphical user interface.

(Windows only) Do I need both git bash and GitKraken? Which one should I use? You don't need both software together, as GitKraken doesn't depend on git bash. We recommend you to use GitKraken, as that will be the software we use to cover how to do anything in class.
Why do I get a permission denied error when I try to install a new library (flask, requests, tweepy) required by the course using pip?

Mac/Linux: This happens since Python is pre-installed into the operating system as mentioned before, which means any library code pip tries to install may end up in folders that require a higher permission level than the pip command has. It's a security mechanism to prevent rogue files being installed onto critical parts of your computer.

To overcome the issue, if you haven't already, I recommend you install Python 3.6 using the Mac OS X installer here.

If you're using a Linux-based OS... then try installing Python 3.6 by setting up pyenv instead.


Windows: Search for the Powershell or cmd in your start menu. Then, right click the result to start the shell as an administrator and try running the commands again. This is relatively safe since we are installing trusted libraries and Windows does not depend on Python like Mac/Linux computers do.

Flask

Question Answer
Why do I get an error page with the following message on the page (along with a page full of traceback) when I visit localhost:5000/...? 
                      
builtins.NameError
NameError: name 'render_template' is not defined
You may have forgotten to import the render_template function from Flask. You can fix that adding render_template at the end of the Flask import's comma separated list.
Why do(es) the HTML template(s) I tried to load using render_template not showing up in the browser when I visit localhost:5000/...? Double check where you saved your HTML files: it needs to be inside the templates folder, otherwise Flask won't be able to find (and serve) them to back to your browser.
Why do(es) the CSS, JavaScript and/or image assets I've included not showing up in the browser when I visit localhost:5000/...?

Firstly, check that you have saved them inside the static folder, otherwise Flask won't be able to find and load them onto your page.

The next thing to pay attention to is the spacing between the {{ }} used to output the location of the assets. There should be no spaces between the double curly braces/brackets.

If that's not the problem, then check that you're using single quotes in the arguments of the url_for function call. This is important since using double quotes in the url_for function call arguments would close off the double quote we used for the src or href attribute in our HTML for including our assets.

Why do I get the following error (together with a long trackback above) when I try to run my Flask app in the terminal? 

AssertionError: View function mapping is overwriting an existing endpoint function: say_hello

You most likely have defined functions with the same name twice or more! Since function names in the same Python file have to be unique, the error is telling you that you are replacing what you've written already, which is probably not what you want!

To fix this error, simply rename your functions so that they all have different names.

Why do some parts of my templates code show up in my browser when I try to visit my app in the browser?

You have probably tried to "visit" the app by double clicking the HTML file directly in File Explorer/Finder. This is not the right way to visit your app since your browser won't understand the additional template syntax that's not really part of HTML!

The correct way to visit your app is by typing localhost:5000/..., as this ensures your template HTML will be run through the template engine which does know how to interpret the additional syntax and so will process and remove them before they are returned to the browser!
